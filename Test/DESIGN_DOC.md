# 设计文档 — 迷你游戏 (Roll-a-Ball)

> **项目名称**: 迷你游戏  
> **游戏类型**: 3D 物理收集 / 逃避追击  
> **目标平台**: Windows PC  
> **操作方式**: 键盘 (WASD / 方向键)  
> **文档日期**: 2026-07-10

---

## 1. 游戏概述

### 1.1 一句话描述

控制一个物理球体在有限场地中收集所有金色方块，同时躲避红色敌人 AI 的追击。

### 1.2 核心循环

```
开始 → 移动球体 → 收集金色方块 → (躲避/利用障碍物阻挡敌人)
                                        ↓
                              收集满 4 个 → 胜利
                              被敌人追上 → 失败
```

### 1.3 目标受众

- Unity 学习者 / 入门开发者
- 适合作为物理 + NavMesh AI 的教学示例项目

---

## 2. 核心玩法设计

### 2.1 玩家角色

| 属性 | 值 |
|---|---|
| 外观 | 青色球体 |
| 移动方式 | 物理力驱动 (AddForce) |
| 移动速度 | 力度 = 10 (受质量与摩擦影响) |
| 碰撞体 | SphereCollider |
| 物理 | Rigidbody (受重力影响) |

**操作感设计**: 使用力而非直接位移，球体有惯性、加速和减速过程，操控具有物理真实感。

### 2.2 收集机制

| 属性 | 值 |
|---|---|
| 外观 | 金色旋转方块 (三轴非均匀旋转) |
| 数量 | 4 个 |
| 碰撞体 | BoxCollider (Trigger) |
| 收集方式 | 玩家球体触碰即收集 |
| 收集效果 | 对象禁用 (SetActive false) |

### 2.3 敌人 AI

| 属性 | 值 |
|---|---|
| 外观 | 红色方块 |
| 移动方式 | NavMeshAgent 自动寻路 |
| 速度 | 2.5 (低于玩家，保证可逃脱) |
| 行为模式 | 持续追踪玩家当前位置 |
| 致命条件 | 与玩家发生物理碰撞 |

**速度平衡**: 敌人速度 (2.5) 低于玩家力度 (10)，确保玩家有逃跑空间，但障碍物和场地限制使得逃避并非毫无策略。

### 2.4 动态障碍物系统

| 属性 | 值 |
|---|---|
| 外观 | 黑色小方块 |
| 物理特性 | 可推动 (mass = 0.1)，受重力影响 |
| NavMesh 影响 | 静止后 carving NavMesh (阻挡敌人路径) |
| 场景数量 | 3 组，每组 4 个，共 12 个 |

**策略深度**: 玩家可以推动黑色方块构建临时屏障，阻挡敌人前进路径。方块静止后会在 NavMesh 上挖洞，迫使敌人重新规划路线。这是核心的策略玩法层。

### 2.5 静态障碍物

| 类型 | 描述 |
|---|---|
| 竖直墙 | 2 个垂直屏障，限制视线和路径 |
| 倾斜坡道 | 2 个 -18° 倾斜面 |
| 大型斜面 | 1 个多角度倾斜障碍 |

**作用**: 限制玩家与敌人的直线距离，增加地形复杂度，使 NavMesh 寻路更有意义。

---

## 3. 胜负条件

### 3.1 胜利条件

- 收集全部 4 个金色方块
- 触发"胜利!"提示
- 自动销毁所有敌人

### 3.2 失败条件

- 敌人物理碰撞玩家
- 销毁玩家球体
- 显示"失败!"提示

### 3.3 无平局机制

游戏必然以胜利或失败结束（敌人持续追踪，不会停止）。

---

## 4. 关卡设计

### 4.1 场地规格

| 参数 | 值 |
|---|---|
| 场地形状 | 正方形 (20 × 20) |
| 地面 | Plane 网格，Scale (2, 1, 2) |
| 围墙 | 4 面高 2 的墙体 |
| 摄像机视角 | 45° 俯视斜角 (0, 10, -10) |

### 4.2 物品布局

```
           N (z=+10)
    ┌─────────────────────┐
    │  PU2        PU0     │
    │                PU3  │
    │   [墙]         [E]  │
    │        [坡道]        │
    │   [墙]    [斜面]     │
    │                     │
    │  PU1        ●P      │
    │            (0,0)    │
    │  [DB堆]     [DB堆]  │
    │                     │
    │        [DB堆]       │
    └─────────────────────┘
    S (z=-10)

    PU = PickUp    P = Player    E = Enemy
    DB = DynamicBox 堆
```

### 4.3 收集品位置

| 编号 | 世界坐标 | 距离玩家 |
|---|---|---|
| PickUp | (4.0, 0.5, 0) | ~4.0 |
| PickUp (1) | (0.02, 0.5, -5.05) | ~5.05 |
| PickUp (2) | (-4.30, 0.5, 0.03) | ~4.30 |
| PickUp (3) | (-0.17, 0.5, 4.13) | ~4.13 |

收集品分布在场地四角方向，玩家需要规划路线。

### 4.4 敌人起始位置

| 坐标 | 距离玩家 |
|---|---|
| (-7.91, 0.5, 6.96) | ~10.5 |

敌人从左上角出生，距玩家约 10.5 单位，给予初始反应时间。

## 5. UI 设计

### 5.1 Canvas 配置

| 参数 | 值 |
|---|---|
| 渲染模式 | Screen Space Overlay |
| CanvasScaler | 响应式缩放 |
| GraphicRaycaster | 启用 |

### 5.2 UI 元素

| 元素 | 位置 | 内容 | 样式 |
|---|---|---|---|
| CountText | 左上角 | "得分: N" | 白色，25px |
| WinText | 屏幕中心 | "胜利!" / "失败!" | 黑色，25px |

### 5.3 UI 状态流

```
游戏开始 → CountText: "得分: 0", WinText: 隐藏
  ↓
收集物品 → CountText: "得分: N"
  ↓
N = 4 → WinText: "胜利!" (显示), Enemy 销毁
  ↓
碰敌人  → WinText: "失败!" (显示), Player 销毁
```

## 6. 视觉设计

### 6.1 色彩方案

| 对象 | 颜色 | 色值 (RGB) | 语义 |
|---|---|---|---|
| 玩家 | 青色 | (0, 0.86, 1) | 友好、玩家 |
| 敌人 | 红色 | (1, 0, 0) | 危险、敌对 |
| 收集品 | 金色 | (1, 0.78, 0) | 奖励、目标 |
| 墙壁/地面 | 深灰 | (0.31, 0.31, 0.31) | 中性环境 |
| 动态障碍 | 黑色 | (0, 0, 0) | 障碍、未知 |

**色彩对比设计**: 玩家(青) vs 敌人(红)形成冷暖对比；金色收集品在灰色环境中醒目；黑色障碍物与浅色地面有明暗对比。

### 6.2 材质风格

- 全部使用 Standard Shader，无贴图，纯色材质
- Metallic = 0（非金属质感）
- Smoothness 差异化：玩家球体最光滑 (0.75)，收集品最哑光 (0.25)
- 光照：单方向光，50° 仰角，硬阴影

### 6.3 动画

| 对象 | 动画类型 | 参数 |
|---|---|---|
| 收集品 | 脚本旋转 | (15, 30, 45)°/s |
| 玩家 | 物理滚动 | 自然滚动（由 Rigidbody 模拟） |
| 敌人 | NavMesh 移动 | 平滑追踪 |

无 Animation Clip 或 Animator Controller，所有运动由脚本或物理引擎驱动。

---

## 7. 摄像机设计

### 7.1 视角

- **类型**: 固定偏移跟随摄像机
- **角度**: 45° 俯视斜角
- **偏移**: (0, 10, -10) — 摄像机始终位于玩家后上方

### 7.2 设计意图

- 俯视角度让玩家看到全场，便于规划收集路线和判断敌人位置
- 固定偏移保持视角稳定，不引入眩晕感
- 使用 `LateUpdate` 确保摄像机平滑跟随

---

## 8. 难度平衡分析

### 8.1 速度对比

| 对象 | 速度 | 备注 |
|---|---|---|
| 玩家 | 力度 10 (加速度受物理影响) | 可快速加速 |
| 敌人 | NavMesh 速度 2.5 | 固定速度 |

### 8.2 平衡评估

| 因素 | 影响 |
|---|---|
| 玩家速度 > 敌人速度 | 玩家可逃脱追击 |
| 4 个收集品需全部收集 | 玩家不能一直逃跑，必须接近危险区域 |
| 动态障碍物 | 玩家可主动改变地形，增加策略 |
| 静态障碍物 | 限制逃跑路线，增加紧张感 |
| 场地有限 (20×20) | 无法无限逃跑 |
| 敌人无冷却 | 持续压力 |

### 8.3 建议改进方向 (未实现)

- 增加多敌人或随时间增加敌人速度
- 收集品分布增加随机性
- 增加计时挑战
- 增加音效反馈
- 增加关卡进度系统

## 9. 输入设计

| 按键 | 功能 |
|---|---|
| W / ↑ | 向前 (Z+) |
| S / ↓ | 向后 (Z-) |
| A / ← | 向左 (X-) |
| D / → | 向右 (X+) |

使用 Unity 旧版 Input Manager 的 `Input.GetAxis("Horizontal")` 和 `Input.GetAxis("Vertical")`，支持键盘和手柄模拟摇杆。

## 10. 技术约束与已知限制

| 约束 | 说明 |
|---|---|
| 无音频系统 | 缺少背景音乐、音效反馈 |
| 无动画系统 | 无 Animation Clip / Animator |
| 无持久化 | 无存档，每次重启重新开始 |
| 无菜单系统 | 无主菜单、暂停、重新开始按钮 |
| 单场景 | 仅一个游戏场景 |
| 旧版输入 | 使用旧版 Input Manager |
| 无后期处理 | 无 Post-processing Stack |
