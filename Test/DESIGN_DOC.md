# 设计文档 — 迷你游戏 (Roll-a-Ball)

> **项目名称**: 迷你游戏
> **游戏类型**: 3D 生存 / 战斗 / 升级 / 逃避追击
> **目标平台**: Windows PC
> **操作方式**: 键盘 (WASD / 方向键) + 鼠标 (左键攻击)
> **文档日期**: 2026-07-10 (更新: 2026-07-15)

---

## 1. 游戏概述

### 1.1 一句话描述

控制一个 AI 生成的冒险者角色在有限场地中收集经验方块升级，用火球消灭持续刷新的敌人，管理血量存活。

### 1.2 核心循环

```
开始 → 移动角色 → 收集经验方块升级 / 火球消灭敌人掉落经验
                          ↓
                    敌人持续从屏幕外刷新
                    接触玩家持续扣血
                          ↓
                    血量归零 → 失败面板
                              ↓
                    重新开始 / 退出游戏
```

### 1.3 目标受众

- Unity 学习者 / 入门开发者
- 适合作为 URP + NavMesh AI + VFX Graph 的教学示例项目

## 2. 核心玩法设计

### 2.1 玩家角色

| 属性 | 值 |
|---|---|
| 外观 | AI 生成冒险者角色 (Meshy AI 生成，含骨骼绑定) |
| 移动方式 | 恒定速度 (rb.velocity) |
| 移动速度 | 10 单位/秒 |
| 碰撞体 | CapsuleCollider (Radius=0.5, Height=2) |
| 物理 | Rigidbody (受重力影响，freezeRotation=true 防翻倒和转圈) |
| 动画 | Animator (Idle/Walk/Run 由 Speed 参数驱动) |
| 攻击 | 鼠标左键发射火球 (VFX Graph 特效) |
| 血量 | 100 HP (HealthBar)，敌人接触持续扣血 |
| 经验 | ExpBar，收集 +10 EXP，满 100 升级，每级 maxExp +20 |
| 脚印 | 移动时左右交替生成，2 秒渐隐消失 |

**操作感设计**: 使用 velocity 直接设定速度而非 AddForce，松手即停，操控直接可控。角色自动朝向移动方向（Quaternion.Slerp 平滑转向），行走时留下脚印增强临场感。鼠标左键发射火球消灭敌人，敌人死亡掉落经验方块，形成"消灭→升级→更强"的正反馈循环。

### 2.2 收集机制

| 属性 | 值 |
|---|---|
| 外观 | 金色旋转方块 (三轴非均匀旋转) |
| 来源 | 场景初始 + 敌人死亡掉落 |
| 碰撞体 | BoxCollider (Trigger) |
| 收集方式 | 玩家触碰即收集 (Destroy) |
| 收集效果 | Destroy 对象 + ExpBar.AddExp(10f) + 拾取音效 |

### 2.3 敌人 AI

| 属性 | 值 |
|---|---|
| 外观 | Quaternius 3D 僵尸模型 |
| 移动方式 | NavMeshAgent 自动寻路 |
| 速度 | 2.5 (低于玩家，保证可逃脱) |
| 行为模式 | 持续追踪玩家当前位置 |
| 伤害 | 接触玩家持续扣血 (20 HP/秒) |
| 死亡掉落 | 在死亡位置生成经验方块 |
| 生成方式 | EnemySpawner 屏幕外持续刷新 (最多 30 个，0.5s 间隔) |

**速度平衡**: 敌人速度 (2.5) 低于玩家力度 (10)，确保玩家有逃跑空间。但敌人持续刷新，数量随时间增加，玩家必须主动消灭敌人控制局面。

### 2.4 火球攻击系统

| 属性 | 值 |
|---|---|
| 外观 | VFX Graph 火焰粒子特效 |
| 发射方式 | 鼠标左键，朝角色朝向发射 |
| 飞行速度 | 20 单位/秒 |
| 碰撞体 | SphereCollider (Trigger) |
| 存活时间 | 3 秒 |
| 效果 | 命中 Enemy → 消灭敌人 + 火球自毁 |

**设计意图**: 火球赋予玩家主动攻击能力，增加游戏策略维度。玩家可以选择逃跑或反击，降低纯被动逃避的挫败感。

### 2.5 动态障碍物系统

| 属性 | 值 |
|---|---|
| 外观 | 黑色小方块 |
| 物理特性 | 可推动 (mass = 0.1)，受重力影响 |
| NavMesh 影响 | 静止后 carving NavMesh (阻挡敌人路径) |
| 场景数量 | 3 组，每组 4 个，共 12 个 |

**策略深度**: 玩家可以推动黑色方块构建临时屏障，阻挡敌人前进路径。方块静止后会在 NavMesh 上挖洞，迫使敌人重新规划路线。这是核心的策略玩法层。

### 2.6 静态障碍物

| 类型 | 描述 |
|---|---|
| 竖直墙 | 2 个垂直屏障，限制视线和路径 |
| 倾斜坡道 | 2 个 -18° 倾斜面 |
| 大型斜面 | 1 个多角度倾斜障碍 |

**作用**: 限制玩家与敌人的直线距离，增加地形复杂度，使 NavMesh 寻路更有意义。

## 3. 胜负条件

### 3.1 失败条件

- 敌人接触玩家持续扣血，血量归零 (0 HP)
- 销毁玩家角色
- 弹出游戏结束面板，显示"失败!"
- 游戏暂停 (Time.timeScale = 0)
- 可选择重新开始或退出游戏

### 3.2 无胜利条件

当前版本为生存模式，敌人持续刷新，无终局胜利条件。目标是在尽可能长的时间内存活并升级。

### 3.3 无平局机制

游戏必然以失败结束（敌人持续刷新追踪，不会停止）。

## 4. 关卡设计

### 4.1 场地规格

| 参数 | 值 |
|---|---|
| 场地形状 | 正方形 (20 × 20) |
| 地面 | Plane 网格，Scale (2, 1, 2) |
| 围墙 | 4 面高 2 的墙体 |
| 摄像机视角 | 45° 俯视斜角 (0, 10, -10) |

### 4.2 物品布局

```
           N (z=+10)
    ┌─────────────────────┐
    │  PU2        PU0     │
    │                PU3  │
    │   [墙]         [E]  │
    │        [坡道]        │
    │   [墙]    [斜面]     │
    │                     │
    │  PU1        ●P      │
    │            (0,0)    │
    │  [DB堆]     [DB堆]  │
    │                     │
    │        [DB堆]       │
    └─────────────────────┘
    S (z=-10)

    PU = PickUp    P = Player    E = Enemy
    DB = DynamicBox 堆
```

### 4.3 收集品位置

| 编号 | 世界坐标 | 距离玩家 |
|---|---|---|
| PickUp | (4.0, 0.5, 0) | ~4.0 |
| PickUp (1) | (0.02, 0.5, -5.05) | ~5.05 |
| PickUp (2) | (-4.30, 0.5, 0.03) | ~4.30 |
| PickUp (3) | (-0.17, 0.5, 4.13) | ~4.13 |

收集品分布在场地四角方向，玩家需要规划路线。

### 4.4 敌人起始位置

| 坐标 | 距离玩家 |
|---|---|
| (-7.91, 0.5, 6.96) | ~10.5 |

敌人从左上角出生，距玩家约 10.5 单位，给予初始反应时间。

## 5. UI 设计

### 5.1 Canvas 配置

| 参数 | 值 |
|---|---|
| 渲染模式 | Screen Space Overlay |
| CanvasScaler | 响应式缩放 |
| GraphicRaycaster | 启用 |

### 5.2 UI 元素

| 元素 | 位置 | 内容 | 样式 |
|---|---|---|---|
| HealthBar | 屏幕底部 | 血量条 (anchorMax.x 驱动) | 红色填充 |
| ExpBar | 屏幕底部 | 经验条 + 等级文本 "Lv. N" | 蓝色填充 |
| GameOverPanel | 屏幕中心 | 失败提示 + 重新开始/退出按钮 | 半透明背景面板 |

### 5.3 UI 状态流

```
游戏开始 → HealthBar: 100%, ExpBar: Lv.1, 0%, GameOverPanel: 隐藏
  ↓
收集经验 → ExpBar.AddExp(10) → 经验条增长
  ↓
满 100 EXP → Lv++ → maxExp += 20 → 播放升级音效
  ↓
敌人接触 → HealthBar.TakeDamage → 血量条减少 + 受伤音效
  ↓
血量归零 → GameOverPanel: "失败!" (显示), Player 销毁, Time.timeScale = 0
  ↓
点击"重新开始" → SceneManager.LoadScene (重置场景)
点击"退出游戏" → Application.Quit()
```

## 6. 视觉设计

### 6.1 色彩方案

| 对象 | 颜色 | 色值 (RGB) | 语义 |
|---|---|---|---|
| 玩家 | AI 生成贴图 | 2048×2048 纹理 | 冒险者角色 |
| 脚印 | 半透明 | 渐隐消失 | 临场感、移动轨迹 |
| 敌人 | 红色 | (1, 0, 0) | 危险、敌对 |
| 收集品 | 金色 | (1, 0.78, 0) | 奖励、目标 |
| 墙壁/地面 | 深灰 | (0.31, 0.31, 0.31) | 中性环境 |
| 动态障碍 | 黑色 | (0, 0, 0) | 障碍、未知 |

**色彩对比设计**: 玩家(青) vs 敌人(红)形成冷暖对比；金色收集品在灰色环境中醒目；黑色障碍物与浅色地面有明暗对比。

### 6.2 材质风格

- AI 生成角色使用贴图材质 (2048×2048 纹理)
- 脚印使用半透明材质，2 秒渐隐消失 (URP _BaseColor)
- 火球使用 VFX Graph 粒子特效
- 其余使用 URP Lit/Unlit Shader，无贴图，纯色材质
- 光照：单方向光，50° 仰角，硬阴影

### 6.3 动画

| 对象 | 动画类型 | 参数 |
|---|---|---|
| 玩家角色 | Animator 状态机 | Idle/Walk/Run 由 Speed (Float) 驱动，Action (Trigger) 触发特殊动作 |
| 火球 | VFX Graph | 火焰粒子特效，3 秒后消失 |
| 收集品 | 脚本旋转 | (15, 30, 45)°/s |
| 敌人 | NavMesh 移动 | 平滑追踪 |
| 脚印 | 脚本渐隐 | 2 秒 alpha 递减至消失 (URP _BaseColor) |

玩家角色动画由 Meshy AI 生成 (Idle/Walk/Run)，Animator Controller 自动创建。

## 7. 摄像机设计

### 7.1 视角

- **类型**: 固定偏移跟随摄像机
- **角度**: 45° 俯视斜角
- **偏移**: (0, 10, -10) — 摄像机始终位于玩家后上方

### 7.2 设计意图

- 俯视角度让玩家看到全场，便于规划收集路线和判断敌人位置
- 固定偏移保持视角稳定，不引入眩晕感
- 使用 `LateUpdate` 确保摄像机平滑跟随

## 8. 难度平衡分析

### 8.1 速度对比

| 对象 | 速度 | 备注 |
|---|---|---|
| 玩家 | 力度 10 (加速度受物理影响) | 可快速加速 |
| 敌人 | NavMesh 速度 2.5 | 固定速度 |

### 8.2 平衡评估

| 因素 | 影响 |
|---|---|
| 玩家恒定速度 10 > 敌人速度 2.5 | 玩家可逃脱追击 |
| 火球攻击 (20 speed) | 玩家可主动消灭敌人，增加策略维度 |
| 4 个收集品需全部收集 | 玩家不能一直逃跑，必须接近危险区域 |
| 动态障碍物 | 玩家可主动改变地形，增加策略 |
| 静态障碍物 | 限制逃跑路线，增加紧张感 |
| 场地有限 (20×20) | 无法无限逃跑 |
| 敌人无冷却 | 持续压力 |

### 8.3 建议改进方向 (未实现)

- 增加多敌人或随时间增加敌人速度
- 收集品分布增加随机性
- 增加计时挑战
- 增加音效反馈
- 增加关卡进度系统

## 9. 输入设计

| 按键 | 功能 |
|---|---|
| W / ↑ | 向前 (Z+) |
| S / ↓ | 向后 (Z-) |
| A / ← | 向左 (X-) |
| D / → | 向右 (X+) |
| 鼠标左键 | 发射火球 |

使用 Unity 旧版 Input Manager 的 `Input.GetAxis("Horizontal/Vertical")` 控制移动，`Input.GetMouseButtonDown(0)` 控制攻击。

## 10. 技术约束与已知限制

| 约束 | 说明 |
|---|---|
| 无背景音乐 | 有 SFX 音效但无 BGM |
| 无持久化 | 无存档，每次重启重新开始 |
| 单场景 | 仅一个游戏场景 |
| 旧版输入 | 使用旧版 Input Manager |
| 无后期处理 | 无 Post-processing Stack |
| AI 生成角色质量 | 角色模型由 Meshy AI 生成，细节和风格受限于 AI 能力 |
| 无终局胜利 | 生存模式，无通关条件 |
